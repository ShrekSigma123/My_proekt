🐍 Практическое моделирование

Зачем моделировать вероятности в играх?

Теория вероятностей даёт формулы, но на практике случайные процессы могут вести себя неожиданно. Симуляция на Python позволяет:

  • Проверить теоретические расчёты (закон больших чисел).
  • Оценить редкие события (например, вероятность выпадения трёх дублей подряд).
  • Сравнить разные игровые стратегии (стоит ли рисковать ради крита?).
  • Визуализировать распределение случайных величин.

Ниже — классический пример: симуляция бросков двух кубиков и сравнение с аналитическими вероятностями.

Симуляция бросков двух кубиков

Симулируем 10 000 бросков для проверки закона больших чисел.

import random
import matplotlib.pyplot as plt

num_rolls = 10000
results = {i: 0 for i in range(2, 13)}

for _ in range(num_rolls):
    die1 = random.randint(1, 6)
    die2 = random.randint(1, 6)
    total = die1 + die2
    results[total] += 1

frequencies = {s: results[s] / num_rolls for s in results}

theoretical = {2: 1/36, 3: 2/36, 4: 3/36, 5: 4/36, 6: 5/36, 7: 6/36,
               8: 5/36, 9: 4/36, 10: 3/36, 11: 2/36, 12: 1/36}

print("Сумма | Эксперимент | Теория")
for s in range(2, 13):
    print(f"{s:4d}   | {frequencies[s]:7.4f} | {theoretical[s]:.4f}")

# График сравнения
plt.bar(range(2,13), [frequencies[s] for s in range(2,13)], alpha=0.6, label='Эксперимент')
plt.plot(range(2,13), [theoretical[s] for s in range(2,13)], 'ro-', label='Теория')
plt.xlabel('Сумма очков'); plt.ylabel('Вероятность')
plt.legend(); plt.show()

Частота 7 ≈ 16.67%. Отклонения уменьшаются с числом бросков. Чем больше итераций, тем ближе экспериментальные частоты к теоретическим (закон больших чисел).

Моделирование критических ударов

Представим, что у персонажа 20% шанс крита и множитель урона ×2. С помощью симуляции можно оценить, как часто серия из 10 ударов даст средний урон ниже ожидаемого.

import random

def simulate_crits(n_attacks=10000, crit_chance=0.2, crit_mult=2, base_damage=100):
    total_damage = 0
    crits = 0
    for _ in range(n_attacks):
        if random.random() < crit_chance:
            total_damage += base_damage * crit_mult
            crits += 1
        else:
            total_damage += base_damage
    avg_damage = total_damage / n_attacks
    print(f"Критов: {crits}/{n_attacks} ({crits/n_attacks:.1%})")
    print(f"Средний урон: {avg_damage:.2f} (теория: {base_damage * (1 + crit_chance*(crit_mult-1)):.2f})")

simulate_crits()

Результат обычно близок к 120 среднего урона. Это подтверждает формулу матожидания, которую мы разбирали на странице про криты.

Оценка вероятности выпадения легендарного предмета

В гача-играх часто стоит задача: «Сколько попыток нужно, чтобы получить предмет с шансом 1%?» Моделирование даёт распределение количества попыток.

import random
import matplotlib.pyplot as plt

def until_success(p=0.01):
    attempts = 0
    while True:
        attempts += 1
        if random.random() < p:
            return attempts

# Симуляция 10 000 игроков
results = [until_success(0.01) for _ in range(10000)]

print(f"Среднее число попыток: {sum(results)/len(results):.1f} (теория: 100)")
print(f"Медиана: {sorted(results)[len(results)//2]}")
print(f"Максимум: {max(results)}")

plt.hist(results, bins=50, density=True, alpha=0.7)
plt.xlabel('Число попыток')
plt.ylabel('Доля игроков')
plt.title('Распределение попыток до выпадения легендарки (шанс 1%)')
plt.show()

Несмотря на среднее 100, многие игроки получат предмет раньше, но некоторые потратят более 300 попыток — это демонстрация «хвоста» распределения.

Закон больших чисел на практике

Чем больше испытаний, тем ближе выборочное среднее к математическому ожиданию. Следующий код показывает, как сходится средний урон при увеличении числа ударов.

import random
import matplotlib.pyplot as plt

cumulative_avg = []
total = 0
for i in range(1, 5001):
    damage = 100 if random.random() < 0.8 else 200
    total += damage
    cumulative_avg.append(total / i)

plt.plot(cumulative_avg, label='Скользящее среднее')
plt.axhline(y=120, color='r', linestyle='--', label='Теоретическое EV=120')
plt.xlabel('Количество ударов')
plt.ylabel('Средний урон')
plt.legend()
plt.show()

График наглядно показывает, как случайные колебания постепенно затухают, и среднее стабилизируется около 120.

Симуляция стратегий в «Камень, ножницы, бумага»

Можно проверить, насколько эффективна случайная стратегия против предсказуемого противника.

import random

def play(strategy1, strategy2, rounds=1000):
    wins1 = 0
    moves = ['rock', 'paper', 'scissors']
    for _ in range(rounds):
        m1 = strategy1()
        m2 = strategy2()
        if (m1 == 'rock' and m2 == 'scissors') or \
           (m1 == 'scissors' and m2 == 'paper') or \
           (m1 == 'paper' and m2 == 'rock'):
            wins1 += 1
    return wins1 / rounds

def random_strategy():
    return random.choice(['rock', 'paper', 'scissors'])

def always_rock():
    return 'rock'

print(f"Случайный против always_rock: {play(random_strategy, always_rock):.2%} побед")
print(f"Случайный против случайного: {play(random_strategy, random_strategy):.2%} побед")

Случайная стратегия против «всегда камень» выигрывает около 33% (только когда выпадает бумага), а против случайного — около 33% побед, 33% поражений и 33% ничьих.

Заключение

Моделирование на Python — мощный инструмент для игрового дизайна и анализа. Оно позволяет:

  • Проверить баланс вероятностей до реализации в коде игры.
  • Обнаружить редкие, но критичные сценарии (например, серию из 10 промахов подряд).
  • Обучиться интуитивному пониманию случайности.

Все приведённые примеры можно запустить в любой среде Python с установленными библиотеками random и matplotlib. Экспериментируйте, меняйте параметры и наблюдайте, как ведут себя вероятности!
